PRIMARY OBJECTIVE
Turn every valid NFC tap into a loyalty, storytelling, CRM and repeat-purchase moment.

nexID must not be “just anti-counterfeit”. It must become:
1. Authentication.
2. Digital Product Passport.
3. Consumer loyalty engine.
4. Experience booking layer.
5. Winery / brand CRM.
6. Gamified data capture.
7. AI-powered campaign generator.
8. Reseller-ready SaaS module for wineries, premium events, perfumes, cosmetics, luxury goods and retailers.

This must work especially well for:
- Mendoza wineries
- Argentine wineries
- LATAM premium brands
- Premium events
- Perfumes
- Cosmetics and creams
- Luxury retail
- Export brands selling in Europe, US, Brazil, Chile, Uruguay, Mexico and global markets

Do not build generic gamification. Build a serious, enterprise-ready growth module that a winery, event producer, perfume brand or premium retailer would actually pay for.

NON-NEGOTIABLE RULES
1. Loyalty must be tied to verified product events.
   - Valid tap can unlock points, badges, quests, rewards and experiences.
   - Replay, revoked, broken, tampered or invalid taps must never grant points.
   - Suspicious taps can create “report problem” flow, ticket, risk alert and CRM event.

2. No fake rewards.
   - Every reward must have a real state: available, locked, claimed, redeemed, expired, out_of_stock, cancelled.
   - Demo mode may use simulated rewards, but UI must clearly mark demo/simulated.

3. No dark patterns.
   - Do not trick users into sharing data.
   - Quizzes and profile data capture must be transparent and opt-in.
   - Consent must be stored and revocable.

4. Alcohol-sensitive verticals.
   - For wine/alcohol experiences, implement optional age gate.
   - Rewards for alcohol must be configurable by tenant and region.
   - Avoid gambling or chance-based mechanics.
   - Do not implement lottery/raffle mechanics unless explicitly enabled and legally configured.

5. Privacy.
   - Store consumer profile data minimally.
   - Location must be approximate by default.
   - Exact location requires explicit permission.
   - Public mobile UX must not expose sensitive anti-fraud internals.

6. Multi-tenant.
   - A tenant owns its own loyalty programs, rewards, campaigns, consumer segments and analytics.
   - Superadmin can view portfolio-level benchmarks but must respect tenant boundaries.
   - Resellers can manage assigned tenants only.

7. Internationalization.
   - Default: es-AR.
   - Also support pt-BR and en.
   - Avoid hardcoded strings. Use existing i18n if present.

PHASE 2 — CORE LOYALTY ENGINE
Implement a single service layer for loyalty.
Suggested files:
- packages/domain/src/loyalty/*
- apps/api/src/services/loyalty/*
- or existing equivalent

Required services:

1. evaluateLoyaltyForTap(input)
Called after every tap event is recorded.
Input:
- tapEvent
- validationResult
- tenant
- tag
- productPassport
- loyaltyProgram

Rules:
- valid tap may award points and update challenge progress.
- replay/tampered/revoked/broken/invalid must not award points.
- repeated valid taps from same member + same tag within cooldown must not farm points.
- anonymous visitors can see locked rewards but must enroll to claim.
- every award must use idempotencyKey.

2. awardPoints(input)
- creates PointsLedger row
- updates LoyaltyMember.pointsBalance and lifetimePoints
- supports positive and negative deltas
- prevents duplicate award for same idempotencyKey

3. evaluateBadges(input)
- checks Badge criteria
- creates MemberBadge if not already earned

4. evaluateChallenges(input)
- updates ChallengeProgress
- marks completed
- allows claim

5. redeemReward(input)
- validates member eligibility
- validates stock
- validates points balance
- deducts points through ledger
- creates RewardRedemption
- returns redemption code

6. checkTierUpgrade(input)
- upgrades member tier based on lifetimePoints, valid taps and rules
- emits event / audit log

7. createReferral(input)
- generates unique referral code
- attributes signups to referrer
- awards only after qualification event

8. bookExperience(input)
- validates capacity
- creates booking
- can be paid/points/free depending policy
- supports check-in via valid NFC tap

9. generatePersonalizedRecommendations(input)
- if AI provider exists: use scoped tenant data
- if no AI provider: deterministic rule-based fallback
- must not hallucinate rewards or offers

10. loyaltyFraudGuard(input)
Detects:
- replay trying to claim points
- too many taps in short period
- same eventId used multiple times
- impossible geo velocity
- same IP/device farming many accounts
- redemption abuse
Blocks/reverses points where appropriate and creates admin alert.
