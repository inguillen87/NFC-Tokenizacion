   - trigger a real download/export

   All tenant metrics, heatmaps, maps and feed data must derive from persisted events.

   - optional POLYGON_CHAIN_ID

5. Localization readiness.
   Default locale: es-AR.
   Prepare pt-BR and en using existing i18n if present. If no i18n exists, create a minimal dictionary module. Do not scatter hardcoded copy.

6. Privacy/geo rule.
   For final-user tap location:
   - default to approximate IP geolocation or existing backend geo method
   - ask explicit browser geolocation permission only when needed
   - store rounded lat/lng or geohash unless exact GPS is explicitly consented
   - never block authentication if geolocation is unavailable
   Winery/bodega address coordinates can be exact because they are business locations.

PHASE 4 — SUN VALIDATION AND EVENT RECORDING
Find existing /sun validation. Wrap it with event recording. Do not replace cryptography.

Required behavior:
1. Existing valid SUN/SDM validation keeps working.
2. Replay detection:
   - repeated same uid + readCounter + cmac/piccData combination must produce REPLAY_SUSPECT or blocked_replay.
   - mobile passport blocks ownership, warranty and tokenization actions for replayed taps.
   - admin shows replay as risk event.
