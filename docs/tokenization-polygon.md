# nexID Polygon Amoy Tokenization Runbook

This runbook keeps the demo blockchain layer practical: SUN validation stays server-side, the chain receives only a private hash of the chip UID, and every physical product can receive a demo certificate / digital twin NFT.

## Current target

- Network: Polygon Amoy testnet
- Chain ID: `80002`
- Gas token: `POL`
- Public explorer: `https://amoy.polygonscan.com`
- Public RPC for wallet demos: `https://rpc-amoy.polygon.technology/`
- Backend RPC can use Polygon, Alchemy, QuickNode, dRPC or another reliable provider.

The in-app wallet card already knows Amoy as:

```ts
chainId: "0x13882"
chainName: "Polygon Amoy"
nativeCurrency: { name: "POL", symbol: "POL", decimals: 18 }
```

## Contract

Contract: `apps/api/contracts/NexidTraceabilityNFT.sol`

What it does:

- ERC-721 digital twin: `Nexid Digital Twin` / `NXDT`.
- `mintWithChipHash(to, chipUidHash, tokenUri, assetRef)` mints one token per private chip hash.
- Duplicate chip hashes are blocked by the contract.
- `updateTraceability(tokenId, traceabilityUri)` lets the backend update a traceability pointer without exposing chip secrets.
- Compatible with OpenZeppelin 5 and Hardhat compiler target `cancun`.

Compile:

```bash
cd apps/api
npm run contracts:compile
```

## Deploy to Amoy

Required env before deploy:

```bash
POLYGON_RPC_URL=https://polygon-amoy.g.alchemy.com/v2/YOUR_KEY
POLYGON_MINTER_PRIVATE_KEY=0xSERVER_MINTER_PRIVATE_KEY
POLYGON_DEPLOY_OWNER=0xSERVER_MINTER_OR_OWNER_ADDRESS
```

Then:

```bash
cd apps/api
npm run contracts:deploy:amoy
```

The script prints JSON. Copy `address` into:

```bash
POLYGON_CONTRACT_ADDRESS=0xDEPLOYED_CONTRACT
```

## API env for Vercel

Set these on the API project (`api.nexid.lat`) in Production and Preview when you want real Amoy anchoring:

```bash
TOKENIZATION_MODE=polygon
SUN_AUTO_TOKENIZE_ON_VALID_TAP=true
TOKENIZATION_USE_LOCAL_MINTER=true
TOKENIZATION_UID_SALT=replace-with-long-random-secret
TOKENIZATION_METADATA_CID_PREFIX=nexid-metadata
POLYGON_RPC_URL=https://polygon-amoy.g.alchemy.com/v2/YOUR_KEY
POLYGON_MINTER_PRIVATE_KEY=0xSERVER_MINTER_PRIVATE_KEY
POLYGON_CONTRACT_ADDRESS=0xDEPLOYED_CONTRACT
POLYGON_DEFAULT_RECIPIENT=0xDEFAULT_DEMO_WALLET_OR_TENANT_WALLET
```

Keep these server-only. Do not expose the private key as `NEXT_PUBLIC_*`.

Optional executor mode:

```bash
TOKENIZATION_EXECUTOR_URL=
TOKENIZATION_EXECUTOR_SECRET=
```

Use it only if minting is moved out of the Next.js API process into a worker.

## Demo mode vs real Amoy mode

The backend supports three useful stages:

1. Simulated ledger: `TOKENIZATION_MODE=simulated`
   - No RPC or private key required.
   - UI shows tx hash and token ID generated by backend.
   - Best for sales demos when blockchain infra is not ready.

2. Amoy backend mint: `TOKENIZATION_MODE=polygon` and `TOKENIZATION_USE_LOCAL_MINTER=true`
   - SUN valid tap creates or processes a `tokenization_requests` row.
   - Backend calls `apps/api/scripts/mint-on-valid-tap.mjs`.
   - The dashboard tokenization queue shows tx hash, token ID and status.

3. External executor: `TOKENIZATION_MODE=polygon` plus `TOKENIZATION_EXECUTOR_URL`
   - Same queue, but minting is delegated to a hardened worker.
   - Recommended later for production isolation.

## Mint one test product manually

```bash
cd apps/api
POLYGON_RPC_URL=https://polygon-amoy.g.alchemy.com/v2/YOUR_KEY \
POLYGON_MINTER_PRIVATE_KEY=0xSERVER_MINTER_PRIVATE_KEY \
POLYGON_CONTRACT_ADDRESS=0xDEPLOYED_CONTRACT \
POLYGON_DEFAULT_RECIPIENT=0xRECEIVER \

  --uid=04AABBCC1122 \
```

## Metadata

The token URI currently points to:

```text
ipfs://<TOKENIZATION_METADATA_CID_PREFIX>/<bid>/<uid>.json
```

Generate metadata locally:

```bash
node apps/api/scripts/build-token-metadata.mjs \
  --bid=DEMO-2026-02 \
  --uid=04AABBCC1122 \
  --image=ipfs://CID/images/product.jpg \
  --passport_url=https://nexid.lat/sun?... \
  --winery="Demo Bodega" \
  --region="Valle de Uco" \
  --vintage=2022
```

For production, pin metadata and images with a pinning service, then set `TOKENIZATION_METADATA_CID_PREFIX` to the real CID/prefix.

## Safety rules

- Never store SUN keys, raw chip secrets or private keys on-chain.
- Store only `chipUidHash`, token URI and business asset reference.
- Use a dedicated minter wallet with limited funds.
- Keep `POLYGON_MINTER_PRIVATE_KEY` only in API/worker env vars.
- Rotate `TOKENIZATION_UID_SALT` only before a production batch starts. Changing it later changes the hash for the same UID.
